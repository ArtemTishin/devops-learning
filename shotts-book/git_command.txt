🚀 Основной рабочий цикл (Ваш ежедневный DevOps-ритуал)
#####################
#####################
git add . (или git add название_файла)
Пояснение: Подготавливает файлы к сохранению (добавляет их в "индекс" или "область подготовки").
Важно: Точка в конце означает «добавить ВСЕ измененные и новые файлы в текущей папке».
Если забыть эту команду, Git просто не увидит ваши правки.
#####################
git commit -m "Понятное описание изменений"
Пояснение: Создает "снимок" (коммит) состояния вашего проекта на вашем компьютере.
Важно: Флаг -m позволяет сразу в кавычках написать комментарий,
чтобы вы и ваши коллеги понимали, что именно было сделано в этой версии кода.
#####################
git push (или при первом запуске git push -u origin master)
Пояснение: Отправляет (выталкивает) все ваши локальные коммиты с вашего компьютера в облако на сайт GitHub [r--].
Только после этой команды ваш код увидят другие люди.
#####################
#####################
📂 Начало работы и синхронизация проектов
#####################
#####################
git init
Пояснение: Инициализирует новый, пустой Git-репозиторий в текущей папке. Создает скрытую папку .git,
где будет храниться вся история изменений.
Эту команду мы использовали один раз в самый первый раз на работе.
#####################
git remote add origin ССЫЛКА_НА_РЕПОЗИТОРИЙ
Пояснение: Связывает вашу локальную папку на компьютере с пустым репозиторием,
который вы только что создали на сайте GitHub [r--].
origin — это стандартное имя для вашего главного облачного репозитория.
#####################
git clone ССЫЛКА_НА_РЕПОЗИТОРИЙ
Пояснение: Скачивает (клонирует) уже существующий проект со всеми файлами 
и полной историей изменений с GitHub [r--] на ваш новый компьютер
#####################
#####################
#####################
🔍 Диагностика и проверка состояния
#####################
#####################
git status
Пояснение: Показывает текущее состояние репозитория. Самая частая команда для проверки.
Она подсвечивает красным измененные или новые файлы
(которые нужно добавить через git add) и зеленым — файлы, которые уже готовы к коммиту.
#####################
#####################
#####################
🛠️ Наведение порядка и удаление ошибок
#####################
#####################
git rm название_файла
Пояснение: Удаляет файл физически с диска и одновременно сообщает Git,
что этот файл больше не должен отслеживаться.
Мы использовали эту команду, чтобы избавиться от ошибочного файла math.shxn.
 После нее обязательно нужно сделать commit и push, чтобы файл пропал и с сайта GitHub.
